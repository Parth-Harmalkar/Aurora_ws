# Plan 4.1: Spatial Detection Pipeline Integration

## Objective
Enable MobileNet-SSD spatial detection on the OAK-D VPU by extending the existing `camera_node.py`. This will produce 3D object detections with coordinates in the camera optical frame.

## Tasks

<task type="auto">
  <name>Configure Detection Network in Pipeline</name>
  <files>ros2_ws/src/aurora_camera/aurora_camera/camera_node.py</files>
  <action>
    Modify the `CameraNode.__init__` to:
    1. Create a `dai.node.MobileNetSpatialDetectionNetwork`.
    2. Set model blob (using a default MobileNet-SSD blob for OAK-D).
    3. Configure `cam_rgb.preview` to 300x300 (required for MobileNet-SSD).
    4. Link `cam_rgb.preview` and `stereo.depth` to the detection network.
    5. Create an `XLinkOut` for detection results ("det").
  </action>
  <verify>Node launches without DepthAI pipeline errors.</verify>
  <done>VPU initializes detection network and starts streaming detections.</done>
</task>

<task type="auto">
  <name>Publish 3D Detections</name>
  <files>ros2_ws/src/aurora_camera/aurora_camera/camera_node.py</files>
  <action>
    Implement processing for the "det" queue in `timer_callback` or a dedicated loop:
    1. Retrieve `spatialDetections` from the "det" queue.
    2. Convert detections to `vision_msgs/msg/Detection3DArray`.
    3. Set `frame_id` to `camera_optical_frame`.
    4. Publish to `/camera/detections`.
  </action>
  <verify>`ros2 topic echo /camera/detections` shows data when objects are present.</verify>
  <done>3D detections are published in standard ROS 2 format.</done>
</task>

## Success Criteria
- [ ] OAK-D pipeline runs with MobileNet-SSD enabled.
- [ ] `/camera/detections` publishes `vision_msgs/Detection3DArray` at ~10Hz.
- [ ] Detections include valid X, Y, Z coordinates in meters.
